# Поступ — крок до вступу

Прототип платформи цільових зборів на навчання.

🔗 **Живе демо:** https://ianitskyi.github.io/postup/

**Документи проєкту:**
- 📄 Опис для донорів і спонсорів: [українською](PROJECT.uk.md) · [English](PROJECT.en.md)
- 🛠️ [Технічне ТЗ для розробника](TECHNICAL-SPEC.md)
- 🧭 [Критичний розбір і план дій](ACTION-PLAN.md)
- 💰 [Бюджет для грантів](BUDGET.md) (з методологією та помісячним розписом)
- 👥 [Склад команди фонду](TEAM.md)

Це **клікабельний прототип** (демо) соціального стартапу: благодійники цільово фінансують
навчання вступників у ліцензовані державою українські заклади освіти (університети, коледжі,
технікуми). Кошти на вступ ідуть через ескроу-рахунок напряму закладу освіти; регулярна
стипендія на час навчання — напряму студенту щомісяця, без академічних умов, доки він
лишається студентом і кошти стипендіального фонду не вичерпані.

## Як відкрити

Варіант 1 (найпростіший): подвійний клік на `index.html` — сайт відкриється в браузері.

Варіант 2 (як справжній сайт): у терміналі в цій папці виконати
```
python -m http.server 8137
```
і відкрити http://localhost:8137

## Що всередині

| Файл | Сторінка |
|---|---|
| `index.html` | Головна: два входи — «Я вступник» / «Я благодійник», статистика, схема руху коштів |
| `apply.html` | Розділ для вступників: створити збір або податися на стипендію |
| `students.html` | Розділ для благодійників: каталог вступників з фільтрами (галузь, заклад освіти, рівень, соціальний критерій) |
| `student.html?id=…` | Профіль: мотиваційний лист, відео, досягнення, рекомендації, таймлайн вступу, панель донату/підписки |
| `scholarships.html` | Іменні стипендії — перегляд і поповнення (для благодійників) + заснування власної |
| `how-it-works.html` | Механізм окремо для вступників, благодійників і закладів освіти + FAQ |
| `admin.html` | Адмінка: дашборд з графіком, модерація заявок, збори, донати, стипендії, виплати |
| `admin-content.html` | Редактор текстів сайту: ручне редагування маркетингових фраз, FAQ, профілів вступників і стипендій без участі ШІ (лише Chrome/Edge, зберігає прямо у файли на диску) |

Усі дані вигадані й лежать у `js/data.js`. «Донати» та рішення модерації зберігаються
в localStorage браузера (кнопка «Скинути демо-дані» в адмінці все обнуляє).
Бекенду, реальних платежів і бази даних тут немає — це макет для перевірки ідеї.

## Що це демо НЕ робить (і що потрібно для справжнього запуску)

1. **Юридична форма.** Зареєструвати Благодійний фонд (БФ) — саме БФ може законно
   збирати пожертви та переказувати їх закладам освіти (на вступ) або студентам напряму
   (регулярні стипендії). Бухгалтер із досвідом неприбуткових організацій обов'язковий
   (ознака неприбутковості в реєстрі); прямі виплати фізособам варто окремо звірити з
   юристом щодо податкових наслідків для отримувача.
2. **Платежі.** Інтеграція з українським платіжним провайдером:
   LiqPay (ПриватБанк), WayForPay, Fondy або monobank еквайринг.
   Для регулярних підписок усі вони мають «рекурентні платежі».
3. **Ескроу / цільовий рахунок.** Формально це окремий банківський рахунок фонду
   з внутрішнім цільовим обліком по кожному збору. Домовленість із банком-партнером —
   ключова перевага для довіри; почніть переговори з банками, що мають програми для НГО.
4. **Верифікація.** Доступ до перевірки статусів вступу — через відкриті дані ЄДЕБО
   (vstup.edbo.gov.ua) можна автоматично підтверджувати подані заяви та зарахування.
5. **Справжній бекенд.** База даних, акаунти, завантаження документів і відео,
   email-розсилки. Розумний перший стек: Next.js + Supabase/PostgreSQL, або
   замовити MVP розробнику, показавши цей прототип як ТЗ.

## З чого житиме платформа (чесна математика)

Комісія на адміністрування — абсолютно нормальна практика для соціальних проєктів:
адміністративні витрати благодійних платформ у світі становлять 5–12% обороту
(GlobalGiving — до 12%, українська dobro.ua — ~6%, Спільнокошт брав ~10%).
Тож 1% — не «забагато», а навпаки, дуже амбітно мало.

Порахуймо: мінімальна команда (координатор, верифікатор, бухгалтер на аутсорсі,
хостинг і підтримка застосунку) — це приблизно 2–3 млн грн на рік. Щоб покрити це
з комісії 1%, річний оборот пожертв має сягати 200–300 млн грн. На старті таких
оборотів не буде, тому модель має бути **гібридною**:

1. **Грант на розробку і перші 1–2 роки роботи** — стартові витрати (див. список
   грантових програм нижче).
2. **Комісія 1%** на адміністрування та розвиток — прозора, зафіксована в правилах
   платформи. Маркетингово сильна: «99 копійок з кожної гривні доходять до мети».
3. **Добровільний внесок благодійника** («чайові платформі», модель GoFundMe):
   при оплаті донор може додати кілька відсотків на роботу сервісу. У GoFundMe це
   основне джерело доходу — люди охоче додають, коли бачать, куди йдуть гроші.
4. **Сервісний внесок корпоративних партнерів** за адміністрування їхніх іменних
   стипендій (компаніям це дешевше, ніж вести програму самим).

Важливо юридично: у статуті БФ прописати, що адміністративні витрати не перевищують
20% (вимога закону про благодійність), і публікувати річний фінзвіт — це
головний аргумент довіри.

## Як рухатися далі (для першого стартапу)

1. **Перевірте попит без коду.** Знайдіть 3–5 реальних вступників і 5–10 потенційних
   донорів. Проведіть перший збір вручну: Google-форма + банка monobank + звіт у
   соцмережах. Якщо ручний процес спрацював — платформу є сенс будувати.
2. **Покажіть цей прототип** закладам освіти (відділи по роботі зі студентами),
   благодійним фондам і потенційним партнерам — збирайте заперечення.
3. **Шукайте грантову підтримку:** Український фонд стартапів, USF Social,
   програми УКФ, House of Europe, EU4Youth — для соціальних стартапів це
   реалістичніший старт, ніж інвестиції.
4. **Партнерство замість конкуренції:** можливо, перший запуск — спільно з наявним
   фондом (наприклад, освітні програми великих БФ), щоб не будувати довіру з нуля.
